# Run A Professional Phase

## Цель

Выполнить стандартную фазу фичи через цикл реализации на основе батчей.

## Предусловия

- Фаза находится в состоянии `TASKLIST_READY`
- `phase`, `plan`, `prd` готовы

## Перед началом

Прочитайте:
- `docs/<TICKET>/phase/<TICKET>/phase-N.md`
- `docs/<TICKET>/plan/<TICKET>-phase-N.md`
- `docs/<TICKET>/prd/<TICKET>-phase-N.prd.md`
- `docs/project/conventions.md`

## Цикл выполнения

```text
/aidd-start-phase N
```

1. Прочитать текущий батч из брифа фазы
2. Сформулировать предложение
3. Дождаться одобрения
4. Реализовать батч
5. Запустить проверки: `/aidd-run-checks`
6. Обновить бриф фазы и таск-лист
7. Показать diff и объяснить
8. Остановиться на значимой границе

## Что такое батч

Не отдельный чекбокс. Батч — это связная инженерная единица:
- Новая доменная сущность + обновление интерфейса + экспорты
- Обработчик состояния + подключение состояния + тесты
- UI-виджет + подключение маршрута + локализация

Полоса Professional: 2-5 связанных задач, если они образуют одну логическую единицу.

## Когда завершать

Когда все элементы выполнения закрыты:

```text
/aidd-complete-phase N
→ reviewer
→ qa
```

## Автоматические ограждения

- **PostToolUse** — каждое редактирование исходного файла запускает автоформатирование
- **PreToolUse** — редактирование блокируется, если нет `.active_ticket`
- **PostCompact** — тикет/фаза/полоса/цель автоматически восстанавливаются при сжатии контекста

## Условия остановки

- Батч завершён
- Обнаружен блокер
- Отклонение от архитектуры
- Обнаружен риск
- Достигнута граница гейта
