# Troubleshooting

## `/hooks` показывает пустой список

- Проверьте, что `.claude/settings.json` существует и содержит валидный JSON
- Перезапустите Claude Code после изменения хуков
- Запустите `/aidd-validate`

## `/agents` не показывает проектных агентов

- Проверьте, что файлы `.claude/agents/*.md` существуют
- Проверьте формат frontmatter (должен содержать `name`, `description`, `tools`)
- Перезапустите Claude Code

## `/mcp` не показывает ожидаемый сервер

- Проверьте, что `.mcp.json` существует в корне проекта
- Убедитесь, что бинарник MCP-сервера установлен локально
- Проверьте команду сервера в `.mcp.json`

## Валидатор упал

```text
/aidd-validate
```

Затем проверьте:
- `docs/project/workflow.md` существует
- `.claude/settings.json` валиден
- Все необходимые агенты и навыки на месте

Валидатор запускается с `context: fork` и `effort: low`.

## Claude забывает контекст после сжатия

Хук PostCompact должен автоматически восстанавливать тикет, фазу, полосу и цель.

Если не работает:
- Проверьте, что `.active_ticket` существует в `docs/<TICKET>/`
- Проверьте, что бриф фазы содержит заголовки `Lane:` и `Goal:`
- Проверьте событие PostCompact в `/hooks`

## Редактирование заблокировано: "No Active Ticket"

Гейт PreToolUse блокирует редактирование исходных директорий без `.active_ticket`.

Исправление:
1. Запустите `/aidd-new-ticket <TICKET>`
2. Или убедитесь, что `docs/<TICKET>/.active_ticket` существует

Защищённые директории настраиваются через `$AIDD_SOURCE_DIRS` (по умолчанию: `lib/*|src/*|packages/*`).

## Дрифт документации фичи

- Проверьте, что заголовок `Workflow Version` соответствует текущей версии (3)
- Проверьте наличие заголовка `Lane`
- Проверьте существование `.active_ticket`
- Валидатор предупреждает о: идее без `Status:`, брифе фазы с 0 отмеченных `[x]` задач

## Автоформатирование не срабатывает

- Проверьте событие PostToolUse в `/hooks`
- Проверьте, что `post-edit-format.sh` Tech Adaptor'а существует
- Хук форматирует только файлы, соответствующие языковому паттерну адаптора
- Убедитесь, что `$CLAUDE_PROJECT_DIR` доступна

## Отсутствует проверка безопасности в Critical

- Полоса должна быть `Critical`
- Файл `security/<TICKET>-phase-N.md` должен существовать
- Без `SECURITY_REVIEW_OK` отправка заблокирована

## Хуки Team Mode не срабатывают

- Проверьте, что установлена `AIDD_TEAM_MODE=1`
- Хуки командного режима (TaskCreated, TaskCompleted, TeammateIdle) подавляются при `AIDD_TEAM_MODE=0`
- Доступно только в Enterprise-тарифе
