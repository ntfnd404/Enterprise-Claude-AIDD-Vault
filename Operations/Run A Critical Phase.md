# Run A Critical Phase

Критическая фаза — это не просто более важная фаза. У неё другой стандарт осторожности.

## Что квалифицируется как Critical

- Аутентификация и авторизация
- Криптографические операции
- Секреты и ключевой материал
- Безопасное хранение и шифрование
- Миграции данных
- Подписание и построение транзакций
- Изменения API-контрактов
- Обработка финансовых данных

## Дополнительные правила

- Батчи меньше, чем в Professional
- Пробелы в проектировании закрываются до реализации
- Предположения, связанные с безопасностью, должны быть явно указаны в плане
- Ревьюер должен подготовить фазу к проверке безопасности

## Поток выполнения

```text
/aidd-start-phase N
→ implement batch
→ /aidd-run-checks
→ /aidd-complete-phase N
→ reviewer
→ security-reviewer
→ qa
```

## Что проверяет security-reviewer

- Отсутствие чувствительного логирования
- Приватный материал не утекает вверх через слои
- Нет небезопасных запасных путей
- Поведение аутентификации/хранилища соответствует плану
- Нет дрифта контрактов, нарушающего границы доверия

## Блокирующее правило

Нельзя переходить к QA, если проверка безопасности вернула блокировку.

## Автоматические ограждения

Те же, что и в Professional:
- **PostToolUse** — автоформатирование исходных файлов
- **PreToolUse** — проверка гейта
- **PostCompact** — восстановление контекста
