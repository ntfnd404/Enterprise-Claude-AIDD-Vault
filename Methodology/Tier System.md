# Система уровней (Tiers)

## Обзор

Методология масштабируется через три уровня. Каждый уровень добавляет больше процесса по мере роста сложности проекта.

## Сравнение

| Возможность | Lite | Standard | Enterprise |
|---|---|---|---|
| **Агенты** | 5 | 7 | 7+ кастомные |
| **Скиллы** | 5 | 8 | 8+ доменные скиллы |
| **Хуки** | 6 | 10 | 12 |
| **Полосы** | Только Professional | Professional + Critical | Trivial + Professional + Critical |
| **Артефактов на фазу** | ~4 | ~7 | ~7+ |
| **Командный режим** | Нет | Нет | Доступен |
| **Security review** | Нет | Только Critical | Critical обязателен |
| **Строгость валидатора** | Базовая | Стандартная | Расширенная |
| **Лог метрик** | Нет | Опционально | Обязательно |
| **ADR** | Опционально | Опционально | Обязательно для архитектурных решений |

## Lite

**Для**: Персональных проектов, прототипов, маленьких фич на 1-2 фазы.

### Что включено
- **5 агентов**: analyst, planner, implementer, reviewer, qa
- **5 скиллов**: new-ticket, start-phase, run-checks, ship-feature, init
- **6 хуков**: InstructionsLoaded, PreToolUse, PostToolUse (adaptor), PostCompact, FileChanged, ConfigChange
- **1 полоса**: только Professional
- **Упрощённая модель гейтов**: `IDEA_READY → PLAN_APPROVED → TASKLIST_READY → IMPLEMENT_STEP_OK → REVIEW_OK → QA_PASS`
- **Упрощённые артефакты**: idea, plan (включает research), brief, qa

### Что исключено
- Нет агента researcher (research включён в planner)
- Нет агента security-reviewer
- Нет отдельных артефактов PRD, research, vision, security review или summary
- Нет скиллов new-phase, complete-phase или validate
- Нет хуков командного режима
- Нет хуков SubagentStart/Stop

### Влияние на токены
Минимальный объём контекста. ~4 артефакта на фичу вместо ~12.

## Standard

**Для**: Продакшн-проектов, 2-5 фаз на фичу, необходима дисциплина ревью и QA.

### Что включено
- **7 агентов**: полный набор
- **8 скиллов**: полный набор
- **10 хуков**: все, кроме командного режима (TaskCreated, TaskCompleted, TeammateIdle)
- **2 полосы**: Professional + Critical
- **Полная модель гейтов**: все 12 гейтов
- **Полный набор артефактов**: idea, vision, tasklist, PRD, research, plan, brief, summary, qa, security
- **Стандартный валидатор**: проверяет файлы, метаданные, хуки

### Что исключено
- Нет командного режима
- Нет полосы Trivial (используйте Professional для мелких изменений)
- ADR опционально

## Enterprise

**Для**: Финтех, крипто, регулируемых продуктов, проектов с несколькими разработчиками.

### Что включено
- **7+ агентов**: полный набор + кастомные доменные агенты
- **8+ скиллов**: полный набор + доменные скиллы
- **12 хуков**: все, включая командный режим
- **3 полосы**: Trivial + Professional + Critical
- **Полная модель гейтов** с обязательным security review для Critical
- **Полный набор артефактов** плюс обязательные ADR и metrics.log
- **Командный режим** доступен для параллельных рабочих потоков
- **Расширенный валидатор**: проверки прогрессии гейтов, обнаружение устаревших фаз

### Дополнительные правила
- Настраиваемый список триггеров Critical (по умолчанию: auth, crypto, storage, API contracts, financial data)
- Обязательный `metrics.log` для аудиторского следа
- Обязательные ADR для архитектурных решений
- Хуки командного режима включены по умолчанию (управляются переменной окружения `AIDD_TEAM_MODE`)

## Конфигурация

Уровень устанавливается при начальной настройке:

```text
/aidd-init --tier standard
```

Хранится как `AIDD_TIER` в `.claude/settings.local.json` или как переменная окружения.

### Повышение уровня

Для повышения с Lite до Standard:

```text
/aidd-init --adopt --tier standard
```

Это добавляет недостающих агентов, скиллы и хуки без нарушения текущей работы.

## Выбор уровня

```
Это персональный/прототипный проект?
  → Lite

Это продакшн-проект с реальными пользователями?
  → Standard

Проект работает с деньгами, секретами или регулируемыми данными?
  → Enterprise
```

Если сомневаетесь, начните со Standard. Повысить уровень можно в любой момент.
