# Ежедневная карта команд

## Команды воркфлоу

| Команда | Когда использовать |
|---|---|
| `/aidd-new-ticket <TICKET>` | Создать рабочее пространство для новой фичи |
| `/aidd-new-phase N` | Создать артефакты фазы |
| `/aidd-start-phase N` | Загрузить контекст, сформулировать предложение батча |
| `/aidd-run-checks` | Форматирование (только изменённые файлы) + анализ + тесты |
| `/aidd-complete-phase N` | Закрыть фазу, направить на ревью |
| `/aidd-validate` | Проверить целостность процессного слоя |
| `/aidd-ship-feature` | Готовность к релизу и синхронизация документации |
| `/aidd-init` | Инициализация нового проекта или обновление воркфлоу |

## Команды рантайма

| Команда | Назначение |
|---|---|
| `/config` | Показать настройки проекта |
| `/agents` | Показать доступных агентов |
| `/hooks` | Показать зарегистрированные хуки |
| `/mcp` | Показать MCP-серверы |


## Команды Superpowers (вспомогательные)

| Команда / практика | Когда использовать | Ограничение |
|---|---|---|
| `/brainstorming` | Уточнение идеи, scope, PRD | Не закрывает `PRD_READY` |
| Adversarial spec review | Проверка PRD/plan на противоречия | Professional: рекомендуется; Critical: обязательно |
| TDD | Реализация поведения внутри батча | Не обязательно для каждого UI/layout изменения |
| Systematic debugging | QA_FAIL, flaky tests, runtime bugs | Сначала root cause, потом fix |
| `/execute-plan` | Исполнение approved batch | Только после `/aidd-start-phase` и approval |
| code-reviewer | Pre-review перед официальным reviewer | Не `REVIEW_OK` |

`/aidd-*` остаются единственными командами воркфлоу и гейтов.

## События хуков (автоматические)

| Событие | Когда срабатывает |
|---|---|
| `InstructionsLoaded` | Загрузка сессии |
| `PostCompact` | Компактификация контекста (восстанавливает тикет/фазу/полосу/цель) |
| `PreToolUse` | Каждое редактирование (проверка гейта) |
| `PostToolUse` | Каждое редактирование исходных файлов (автоформатирование) |
| `FileChanged` | Изменение файлов воркфлоу |
| `ConfigChange` | Изменение настроек/навыков |
| `SubagentStart/Stop` | Жизненный цикл агента |
