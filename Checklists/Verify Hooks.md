# Проверка хуков

Используйте этот чек-лист для диагностики и верификации настроенных хуков Claude Code. Подробности о хуках в контексте фаз см. [[Operations/Run A Professional Phase]] и [[Operations/Run A Critical Phase]].

## Регистрация

- [ ] `/hooks` показывает все зарегистрированные хуки

## Хуки жизненного цикла

- [ ] `InstructionsLoaded` срабатывает при загрузке сессии
- [ ] `PostCompact` восстанавливает контекст (тикет, фаза, полоса, цель)

## Хуки контроля качества

- [ ] `PreToolUse` блокирует правки без гейта (`.active_ticket` обязателен)
- [ ] `PostToolUse` форматирует файлы (если Tech Adaptor настроен)

## Хуки отслеживания изменений

- [ ] `FileChanged` отслеживает изменения воркфлоу-файлов
- [ ] `ConfigChange` отслеживает изменения настроек

## Общие правила

- [ ] Async хуки не блокируют выполнение
- [ ] `statusMessage` используется вместо `systemMessage` где возможно
