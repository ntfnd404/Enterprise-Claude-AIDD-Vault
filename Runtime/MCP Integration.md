# Интеграция MCP

## Что такое MCP

MCP (Model Context Protocol) -- протокол, позволяющий Claude Code использовать внешние инструменты для детерминированных операций. Ключевое преимущество: MCP-вызовы не проходят через рассуждения Claude, что делает их значительно дешевле по токенам.

## Философия

Используйте MCP для детерминированных операций. НЕ используйте MCP как замену для архитектурного мышления, оценки ревью, рассуждений о безопасности или оценки QA.

## Стратегия MCP-First

Основной принцип: используйте MCP-инструменты для format/analyze/test, откатывайтесь к bash только когда MCP недоступен.

| Задача | MCP-инструмент | Фоллбэк |
|---|---|---|
| Форматирование кода | MCP format (стек-специфичный) | CLI-форматтер |
| Статический анализ | MCP analyze (стек-специфичный) | CLI-анализатор |
| Запуск тестов | MCP test (стек-специфичный) | CLI test runner |
| Поиск символов | MCP hover/resolve | Grep + Read |
| Управление зависимостями | MCP pub/npm/go | CLI пакетный менеджер |

### Почему MCP дешевле

Когда Claude вызывает bash-команду, происходит:
1. Claude рассуждает о команде (токены рассуждения)
2. Claude формирует команду (токены вывода)
3. Команда выполняется
4. Claude читает и анализирует результат (токены рассуждения)

Когда Claude использует MCP-инструмент:
1. Вызов идёт напрямую к серверу (минимальные токены)
2. Результат структурирован и предсказуем

Экономия наиболее заметна на повторяющихся операциях: форматирование, анализ, тесты.

## Конфигурация .mcp.json

MCP-серверы настраиваются в файле `.mcp.json` в корне проекта:

```json
{
  "mcpServers": {
    "<server-name>": {
      "command": "<binary>",
      "args": ["<arg1>", "<arg2>"]
    }
  }
}
```

Пример для Dart MCP-сервера:

```json
{
  "mcpServers": {
    "dart": {
      "command": "dart",
      "args": ["tooling-daemon", "--mcp"]
    }
  }
}
```

### Проверка настройки MCP

```text
/mcp
```

Должен показать настроенные MCP-серверы. Если пусто -- проверьте `.mcp.json` и убедитесь, что серверный бинарник установлен.

## Tech Adaptors и MCP

Tech Adaptors предоставляют рекомендованную конфигурацию `.mcp.json` при инициализации проекта. Например:

- **Flutter-Dart** адаптор предоставляет Dart MCP-сервер для форматирования, анализа и работы с символами
- **Node-TypeScript** адаптор может предоставить TypeScript Language Server через MCP
- **Go** адаптор может предоставить gopls через MCP

Tech Adaptor настраивается при запуске `/aidd-init --adaptor <name>`. Стек-специфичные детали конфигурации MCP живут в документации соответствующего Tech Adaptor, а не здесь.

## Конвейер проверок и MCP

Скилл `/aidd-run-checks` выполняет `.claude/aidd-checks.sh`, который генерируется Tech Adaptor при инициализации. Конвейер:

1. **Format** -- только изменённые файлы (используйте `git diff --name-only` для списка файлов)
2. **Analyze** -- статический анализ с нулевой толерантностью
3. **Lint** -- дополнительные правила (зависят от стека)
4. **Test** -- полный набор тестов

MCP-first: попробовать MCP-инструмент, откатиться к CLI при недоступности.

### Форматирование только изменённых файлов

Вместо форматирования всей кодовой базы форматируйте только файлы, изменённые в текущей сессии:

```bash
git diff --name-only --diff-filter=d HEAD | grep '\.<ext>$'
```

Передайте список файлов в параметр `paths` MCP или в CLI-форматтер. Это экономит как время выполнения, так и токены.

## Хук PostToolUse и автоформатирование

Хук PostToolUse (предоставляемый Tech Adaptor) автоматически форматирует исходные файлы после каждого Write/Edit/MultiEdit. Форматирует только файлы, соответствующие паттерну языка адаптора.

Это отдельный механизм от `/aidd-run-checks` -- код остаётся отформатированным в процессе работы, а не только при проверке.

## Оптимизация скилла /aidd-run-checks

Скилл использует:
- `context: fork` -- вывод не загрязняет основной контекст
- `effort: low` -- механическая операция, глубокое рассуждение не требуется

## Экономия токенов от MCP

| Оптимизация | Экономия |
|---|---|
| MCP format вместо bash format | ~100 tok за вызов |
| MCP analyze вместо bash analyze | ~100 tok за вызов |
| MCP test вместо bash test | ~80 tok за вызов |
| Суммарно (MCP + артефакты) | ~450 tok за сессию |

Подробнее об общей экономии см. [[../Methodology/Token Optimization]].
