# Командный режим

## Что это

Командный режим -- параллельные рабочие потоки с несколькими экземплярами Claude. Позволяет одновременно планировать одну фазу, реализовывать другую и проводить QA третьей.

## По умолчанию

Командный режим выключен. Для большинства задач обычных субагентов достаточно.

## Когда включать

Включайте командный режим когда:
- 3+ фазы в работе или готовы к параллельному выполнению
- Фазы не имеют перекрёстных зависимостей
- Существуют отдельные рабочие потоки (один планирует, пока другой реализует)
- Research, review, security review или QA могут выполняться параллельно

## Как включить

Через переменную окружения:

```bash
export AIDD_TEAM_MODE=1
```

Или через `.claude/settings.local.json`:

```json
{
  "env": {
    "AIDD_TEAM_MODE": "1"
  }
}
```

При `AIDD_TEAM_MODE=0` (по умолчанию) хуки командного режима подавляются -- скрипты завершаются сразу.

## Архитектура

```
Оркестратор (основной контекст)
  |-- Агент: Planner(phase N+1)       -- готовит план, пока Implementer работает над N
  |-- Агент: Implementer(phase N)     -- читает phase/ + plan/
  +-- Агент: QA(phase N-1)            -- верифицирует, пока Implementer работает над N
```

Каждый участник команды получает непересекающийся рабочий поток. Оркестратор остаётся в основном контексте и координирует распределение задач.

## Координация

### Общие ресурсы
- Директория `docs/` -- общее пространство артефактов
- `docs/project/` -- конвенции, стайлгайд, рабочий процесс

### Изоляция
- Каждый участник работает в отдельной feature-ветке или на отдельной фазе
- Никакие два агента не пишут в одни и те же файлы
- Все гейты по-прежнему применяются

### Принцип
- Оркестратор распределяет задачи
- Каждый участник работает автономно в рамках своего потока
- При обнаружении отклонения от архитектуры -- немедленная эскалация к оркестратору

## События хуков

Когда `AIDD_TEAM_MODE=1`, срабатывают три дополнительных хука:

| Событие | Назначение | Скрипт |
|---|---|---|
| `TaskCreated` | Advisory: создана новая задача | `team-task-lifecycle.sh created` |
| `TaskCompleted` | Advisory: задача завершена | `team-task-lifecycle.sh completed` |
| `TeammateIdle` | Сигнал: участник свободен, рассмотреть ребалансировку | `team-task-lifecycle.sh idle` |

Все три хука работают в режиме async (неблокирующие, только advisory). Не инъектируют данные в контекст Claude и не стоят токенов рассуждения.

Когда `AIDD_TEAM_MODE=0` (по умолчанию), эти события подавляются -- скрипт проверяет переменную и завершается сразу:

```bash
if [[ "${AIDD_TEAM_MODE:-0}" != "1" ]]; then
  exit 0
fi
```

## Структура в settings.json

```json
{
  "hooks": {
    "TaskCreated": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "bash \"$CLAUDE_PROJECT_DIR/.claude/hooks/team-task-lifecycle.sh\" created",
            "timeout": 10,
            "async": true
          }
        ]
      }
    ],
    "TaskCompleted": [...],
    "TeammateIdle": [...]
  }
}
```

## Доступность по уровням

Командный режим доступен **только в уровне Enterprise**.

| Возможность | Lite | Standard | Enterprise |
|---|---|---|---|
| Командный режим | Нет | Нет | Да |
| Хуки TaskCreated/TaskCompleted/TeammateIdle | Нет | Нет | Да |

В уровне Enterprise хуки командного режима включены по умолчанию и управляются переменной окружения `AIDD_TEAM_MODE`.

См. [[../Methodology/Tier System]] для полного сравнения уровней.

## Когда НЕ использовать командный режим

- Средние фичи без реального параллелизма
- Тесно связанные фазы, где выход одной напрямую подаётся в следующую
- Небольшие команды, где накладные расходы на переключение контекста превышают выгоду от параллелизма
- Фазы полосы Critical с перекрёстными зависимостями по безопасности
